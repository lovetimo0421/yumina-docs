# Yumina 创作者文档设计方案

**日期**: 2026-03-23
**状态**: 已批准

## 目标

为 Yumina 创作一份全面、好看、简单易懂的中文创作者文档。

## 设计决策

| 决策项 | 选择 |
|--------|------|
| 目标读者 | 新手 + 有经验的创作者，两者都照顾 |
| 语言 | 中文 |
| 部署形式 | Markdown 文件放 `docs/creator-guide/`，载体后定 |
| 风格/语气 | 轻松口语化，像朋友教你，用比喻和例子 |
| 实战教程 | 需要，从零到发布的完整案例 |
| 与酒馆对比 | 不提，Yumina 是独立产品 |
| 复杂功能讲解 | 双层结构：🟢简单版 + 🔵详细版 |

### 每章内部结构

```markdown
# 章节标题

> 一句话总结这章讲什么

## 🟢 简单版
（大白话 + 最小例子，500字以内）

## 🔵 详细版
（完整语法、所有字段、高级用法、注意事项）

## 💡 实用例子
（2-3个真实场景的完整示例）
```

## 各章内容规划

### 第一部分：创作之旅

#### `00-welcome.md` — 欢迎来到 Yumina
- Yumina 是什么（一句话）
- 创作者能做什么（能力概览）
- 文档怎么用（教程 vs 参考手册）
- "5分钟感受"：引导去体验精选世界

#### `01-core-concepts.md` — 核心概念速览
- 世界（World）= 完整的互动体验包
- 词条（Entry）= 内容碎片（角色/场景/规则/风格）
- 变量（Variable）= 世界的记忆
- 指令（Directive）= AI 怎么改变世界状态
- 规则（Rule）= 自动化行为
- 组件（Component）= UI 面板
- 渲染器（Renderer）= 聊天消息样式
- 流程图：词条→提示词→AI→指令→变量→规则→渲染

#### `02-tutorial-basic.md` — 普通版教程（以 "No, I am not human" 为蓝本）
- 从空白世界开始
- 写角色词条和开场白
- 创建游戏变量（HP、体力、天数）
- 教 AI 用指令更新状态
- 添加简单 HUD 组件
- 添加世界书词条（NPC关键词触发）
- 测试和发布

#### `02-tutorial-advanced.md` — 高级版教程（以 "壺中の毒 · 大逃杀" 为蓝本）
- 前置：假设已完成普通版
- 多角色世界设计（40个NPC的组织方式）
- 复杂变量结构（JSON嵌套、数组）
- 行为规则链（饥饿衰减、自动死亡、氛围切换）
- 自定义消息渲染器（TSX）
- 全屏游戏面板（角色创建、地图、名册）
- 音频系统（条件触发BGM）
- 调试和优化技巧
