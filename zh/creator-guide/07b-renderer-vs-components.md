# 消息渲染器 vs 自定义组件

> 两种自定义 UI 的方式，看起来很像，但用法完全不同。这篇帮你搞清楚它们的区别，以及什么时候该用哪个。

---

## 一句话总结

- **消息渲染器（messageRenderer）**：改变每条 AI 消息的样子。用在普通聊天模式。
- **自定义组件（customComponents）**：接管整个屏幕，做独立的游戏界面。用在全屏模式。

**两者不会同时生效。** 它们由一个开关控制：`fullScreenComponent`。

---

## 它们的关系

```
fullScreenComponent = false（默认）
  → 玩家看到：普通聊天界面
  → messageRenderer：✅ 生效（每条消息都经过它渲染）
  → customComponents：❌ 不显示（即使你写了代码也看不到）

fullScreenComponent = true
  → 玩家看到：全屏自定义界面（没有聊天框、没有输入框）
  → messageRenderer：❌ 不生效（完全不被调用）
  → customComponents：✅ 占满整个屏幕
```

这不是"两个东西同时存在"——而是**二选一**。开关关着，走消息渲染器路线；开关开着，走自定义组件路线。

---

## 详细对比

| | 消息渲染器 | 自定义组件 |
|--|-----------|-----------|
| **在编辑器哪里设置** | 消息渲染器 标签页 | 进入工作室 → 代码视图 |
| **数量** | 只能有 1 个 | 可以有多个 |
| **什么时候显示** | 普通聊天模式（默认） | 仅全屏模式 |
| **显示位置** | 替换每条消息的渲染 | 占满整个屏幕 |
| **聊天界面** | 正常显示（消息列表 + 输入框） | **完全消失** |
| **能拿到的数据** | 每条消息的 `content`、`role`、`messageIndex` + `useYumina()` | 只有 `useYumina()`（没有单条消息数据） |
| **典型用途** | 气泡对话、状态面板、战斗日志、交互按钮 | 视觉小说全屏界面、完整游戏 UI |
| **全屏模式下** | 不工作 | 工作 |
| **普通模式下** | 工作 | 不工作 |

---

## 消息渲染器：怎么工作

消息渲染器是一段 TSX 代码，**替换默认的 Markdown 渲染**。每当 AI 发了一条消息，引擎不再用纯文字显示，而是把消息内容交给你的代码来决定怎么画。

**它是逐条调用的。** 聊天里有 10 条消息，你的渲染器就会被调用 10 次，每次拿到不同的 `content` 和 `messageIndex`。

### 收到的数据

```tsx
export default function Renderer({
  content,         // 这条消息的文字（已去掉指令）
  role,            // "user" 或 "assistant"
  messageIndex,    // 这是第几条消息（从 0 开始）
  renderMarkdown,  // 内置函数：把 Markdown 转成 HTML
  isStreaming,     // AI 是否正在打字
}) {
  const api = useYumina();  // 还可以拿到游戏状态和交互 API
  // ...
}
```

### 适合做什么

- 在消息上方/下方加状态面板（HP 条、金币显示）
- 把消息改成气泡对话样式
- 在第一条消息上放路线选择按钮
- 在最后一条消息下面放交互输入框
- 战斗日志、带颜色标记的文字
- 任何需要**和聊天消息共存**的 UI

### 关键限制

- **只能有 1 个** — 不能同时用多个渲染器
- **全屏模式下不工作** — 如果开了 `fullScreenComponent: true`，渲染器不会被调用
- **每条消息都调用** — 如果你只想在某条消息上显示内容，需要自己用 `messageIndex` 或 `isLastMsg` 判断

## 自定义组件：怎么工作

自定义组件是**完全独立的 UI 面板**。它不替换消息渲染，而是**接管整个屏幕**——聊天消息列表、输入框、甚至顶部导航栏全部消失，只剩你的组件。

**你必须打开全屏模式才能看到它。** 在编辑器的概览标签页或世界设置中，把 `fullScreenComponent` 设为 `true`。

### 收到的数据

```tsx
export default function MyComponent() {
  const api = useYumina();
  // api.variables     — 游戏状态
  // api.sendMessage() — 发消息（触发 AI 回复）
  // api.setVariable() — 改变量
  // api.executeAction() — 触发行为
  // api.messages      — 所有消息历史
  // api.isStreaming    — AI 是否在打字
  // api.streamingContent — AI 正在打的内容
  // ...
}
```

注意：它**没有** `content`、`role`、`messageIndex`、`renderMarkdown` 这些参数。因为它不是在渲染某条消息——它是一个独立的全屏界面。

### 适合做什么

- 视觉小说全屏界面（背景 + 立绘 + 对话框 + 选项）
- 完整的游戏 UI（地图 + 状态栏 + 物品栏 + 聊天窗口全部自己画）
- 任何不想要默认聊天界面的场景

### 关键限制

- **只在全屏模式下显示** — `fullScreenComponent: false`（默认）时，组件完全不渲染
- **聊天界面消失** — 玩家看不到消息列表和输入框，你需要自己用 `api.sendMessage()` 和 `api.messages` 来处理聊天
- **ESC 会退出世界** — 全屏模式下按 ESC 不是退出全屏，而是离开这个世界

---

## 怎么选？

```
你想做的事 → 用哪个
─────────────────────
在消息上加 HP 条          → 消息渲染器
在消息下加按钮            → 消息渲染器
改变消息的外观（气泡、颜色） → 消息渲染器
做一个全屏视觉小说        → 自定义组件 + fullScreenComponent: true
做一个完整的游戏界面      → 自定义组件 + fullScreenComponent: true
保留聊天界面但加个侧边栏  → 目前不支持（自定义组件只有全屏模式）
```

> **大多数创作者只需要消息渲染器。** 自定义组件是给想要完全掌控界面的高级创作者准备的。如果你不确定用哪个，先用消息渲染器。

---

## 在编辑器里怎么设置

### 消息渲染器

1. 打开编辑器 → 左侧边栏 → **消息渲染器** 标签页
2. 选择「自定义 TSX」
3. 在代码框里写 TSX 代码
4. 底部显示「编译状态：正常」就说明语法没问题
5. 保存，开新会话测试

### 自定义组件

1. 打开编辑器 → 点击顶部「进入工作室」按钮
2. 在工作室的「代码视图」面板里，点击 **+** 添加组件
3. 写 TSX 代码
4. 回到编辑器 → **概览** 标签页 → 确认全屏模式已开启
5. 保存，开新会话测试

## 常见误解

### "我写了自定义组件但看不到"

最常见的原因：**全屏模式没开**。自定义组件只在 `fullScreenComponent: true` 时显示。在编辑器的概览标签页检查这个设置。

### "我开了全屏模式但聊天消失了"

这是正常行为。全屏模式下，自定义组件接管整个屏幕，聊天界面不再显示。如果你想保留聊天，需要在自定义组件里自己用 `api.messages` 和 `api.sendMessage()` 来实现一个聊天界面。

### "我能同时用两个吗？"

在当前版本中，**不能同时让两者对玩家可见**：
- 普通模式 → 只有消息渲染器生效
- 全屏模式 → 只有自定义组件生效

如果你想在全屏游戏界面里嵌入一个聊天窗口，需要在自定义组件里自己实现。

### "它们用同一个 API 吗？"

是的。两者都可以调用 `useYumina()`，拿到完全相同的 API：`sendMessage`、`setVariable`、`executeAction`、`switchGreeting`、`playAudio`、`stopAudio`、`variables`、`messages` 等。

唯一的区别是消息渲染器额外收到**单条消息的数据**（`content`、`role`、`messageIndex`、`renderMarkdown`），而自定义组件没有这些。
