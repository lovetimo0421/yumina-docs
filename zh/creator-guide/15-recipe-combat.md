# 战斗系统

> 做一个带 HP/MP 血条、攻击/防御/逃跑按钮的战斗界面。点击攻击 → 扣敌人血、敌人反击扣你血。HP 归零 → 弹出死亡通知。这篇教你怎么用变量、行为和消息渲染器搭出来。

---

## 你要做的东西

一个完整的回合制战斗系统：

- **HP 和 MP 血条**——实时显示在聊天界面里，玩家一眼就能看到当前状态
- **攻击 / 防御 / 逃跑按钮**——玩家点击后触发不同的行为
- **伤害计算**——攻击时扣敌人 HP，敌人反击扣玩家 HP；防御时减少受到的伤害并恢复 MP
- **胜负判定**——敌人 HP 降到 0 以下弹出「击败敌人！」成就通知；玩家 HP 降到 0 以下弹出「你死了」危险通知
- **战斗氛围切换**——进入战斗时启用战斗氛围知识条目，退出时禁用

### 原理

整个系统的核心是：**按钮触发行为 → 行为修改变量 → 变量变化触发更多行为**。

```
玩家点击「攻击」按钮
  → 触发「attack」行为
  → 行为执行：enemy_hp 减 15，player_hp 减 10（敌人反击）
  → 引擎检测到 enemy_hp 跨过阈值 0
  → 触发「敌人被击败」行为
  → 弹出成就通知，设置 in_combat = false
  → 请求 AI 描述战斗结果
```

这就是 Yumina 行为系统的威力——你不需要写任何代码来处理逻辑，全部用编辑器里的行为规则配置就行。消息渲染器只负责显示血条和按钮。

## 一步步来

### 第 1 步：创建变量

我们需要 4 个变量来追踪战斗状态。

编辑器 → 左侧边栏 → **变量** 标签页 → 逐个点击「添加变量」

**变量 1：玩家 HP**

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 显示名称 | 玩家生命值 | 给你自己看的，方便在变量列表里识别 |
| ID | `player_hp` | 行为规则和消息渲染器代码用这个 ID 来读写 |
| 类型 | 数字 | HP 是数值，需要做加减运算 |
| 默认值 | `100` | 满血开局，最大值 100 |
| 分类 | 属性 | 和角色属性相关的变量放在「属性」分类下 |
| 行为规则 | `不要直接修改这个变量。它由战斗系统的行为规则控制。当前值代表玩家的剩余生命值，最大 100。` | 告诉 AI 不要自作主张改 HP——只有行为规则能改 |

**变量 2：玩家 MP**

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 显示名称 | 玩家魔力值 | 方便识别 |
| ID | `player_mp` | 防御时会恢复 MP |
| 类型 | 数字 | 同样是数值 |
| 默认值 | `50` | 满魔力开局，最大值 50 |
| 分类 | 属性 | 角色属性 |
| 行为规则 | `不要直接修改这个变量。它由战斗系统的行为规则控制。当前值代表玩家的剩余魔力值，最大 50。` | 防止 AI 擅自修改 |

**变量 3：敌人 HP**

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 显示名称 | 敌人生命值 | 方便识别 |
| ID | `enemy_hp` | 攻击时扣这个变量 |
| 类型 | 数字 | 数值运算 |
| 默认值 | `80` | 敌人 80 HP，比玩家低一点 |
| 分类 | 属性 | 角色属性 |
| 行为规则 | `不要直接修改这个变量。它由战斗系统的行为规则控制。当前值代表敌人的剩余生命值，最大 80。` | 防止 AI 擅自修改 |

**变量 4：是否在战斗中**

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 显示名称 | 战斗状态 | 方便识别 |
| ID | `in_combat` | 控制战斗 UI 是否显示、知识条目是否启用 |
| 类型 | 布尔 | 只有「在战斗中」和「不在战斗中」两个状态 |
| 默认值 | `false` | 默认不在战斗中。进入战斗时由行为设为 `true` |
| 分类 | 标记 | 这是一个状态标记，不是数值属性 |
| 行为规则 | `不要直接修改这个变量。它由战斗系统控制。true 表示正在战斗，false 表示不在战斗中。` | 防止 AI 擅自修改 |

::: info 为什么每个变量都要写行为规则？
因为 AI 在生成回复时可以「建议」修改变量。如果你不告诉它别碰，它可能在叙述中自行改变 HP（比如 AI 觉得"玩家被打了一拳"所以自己减 HP）。行为规则字段就是对 AI 的指令——写了之后 AI 就知道这些变量只由系统控制。
:::

### 第 2 步：创建战斗氛围知识条目

这个条目在战斗中启用，让 AI 的回复风格更有战斗感。

编辑器 → **知识库** 标签页 → 新建条目

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 名称 | 战斗氛围 | 给你自己看的 |
| 区域 | 预设 | 预设区的条目每次都会发给 AI |
| 启用 | **否**（关闭开关） | 默认不启用——等进入战斗后行为规则再把它打开 |

内容：

```
[战斗模式]
当前正在进行战斗。请遵守以下规则：
- 用紧张激烈的语气描写战斗场面
- 描写每一次攻击的动作细节：武器的轨迹、命中的部位、造成的效果
- 描写敌人的反应：痛苦、愤怒、恐惧或疯狂
- 根据双方剩余 HP 调整描述的紧迫感——HP 低时描写更加绝望
- 不要自己决定战斗结果，只描述当前回合发生的事情
- 玩家当前 HP：{{player_hp}}/100，MP：{{player_mp}}/50
- 敌人当前 HP：{{enemy_hp}}/80
```

> **为什么默认禁用？** 因为玩家不一定一开始就在战斗。可能先有一段探索、对话的剧情，然后才遭遇敌人。只有当 `in_combat` 变为 `true` 时，行为规则才启用这个条目，AI 才切换到战斗描写模式。
>
> **注意条目里的宏。** `{{player_hp}}`、`{{player_mp}}`、`{{enemy_hp}}` 会在每次构建提示词时被替换为变量的当前值。这样 AI 每次回复都能看到最新的血量数据，写出更准确的战斗描述。

### 第 3 步：创建行为规则

这是整个战斗系统的核心。我们需要 5 个行为，分别处理攻击、防御、逃跑、敌人被击败、玩家死亡。

编辑器 → **行为** 标签页 → 逐个添加行为

#### 行为 1：攻击

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 名称 | 攻击 | 给你自己看的 |
| 触发条件 | 选「动作」→ 动作 ID 填 `attack` | 当消息渲染器按钮调用 `executeAction("attack")` 时触发 |

然后在「执行动作」里依次添加：

| 动作类型 | 设置 | 作用 |
|---------|------|------|
| 修改变量 | `enemy_hp` 减去 `15` | 玩家攻击造成 15 点伤害 |
| 修改变量 | `player_hp` 减去 `10` | 敌人反击造成 10 点伤害 |
| 请求 AI 回复 | *（无需额外设置）* | 让 AI 描述这一回合的攻击过程 |

> **为什么攻击里有两次「修改变量」？** 因为这是简化版的回合制——玩家攻击的同时，敌人也会反击。你当然可以把反击做成单独的行为（用定时器或者额外的触发器），但对于入门配方来说，同时处理更直观。
>
> **为什么最后要「请求 AI 回复」？** 因为变量修改是静默的——玩家只能看到血条变化，但看不到「发生了什么」。请求 AI 回复后，AI 会根据战斗氛围条目写出精彩的战斗描写（"你挥剑劈向巨狼的侧腹……"）。

#### 行为 2：防御

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 名称 | 防御 | 给你自己看的 |
| 触发条件 | 选「动作」→ 动作 ID 填 `defend` | 按钮调用 `executeAction("defend")` 时触发 |

执行动作：

| 动作类型 | 设置 | 作用 |
|---------|------|------|
| 修改变量 | `player_hp` 减去 `5` | 防御时只受 5 点伤害（比攻击时的 10 点少） |
| 修改变量 | `player_mp` 加上 `10` | 防御姿态恢复 10 点魔力 |
| 请求 AI 回复 | *（无需额外设置）* | 让 AI 描述防御的过程 |

> **为什么防御还是会扣血？** 因为防御不是无敌——只是减少伤害。同时恢复 MP 给玩家一个战术选择：冒险攻击 vs 安全防御回蓝。

#### 行为 3：逃跑

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 名称 | 逃跑 | 给你自己看的 |
| 触发条件 | 选「动作」→ 动作 ID 填 `flee` | 按钮调用 `executeAction("flee")` 时触发 |

执行动作：

| 动作类型 | 设置 | 作用 |
|---------|------|------|
| 修改变量 | `in_combat` 设为 `false` | 退出战斗状态，战斗 UI 消失 |
| 禁用知识条目 | 战斗氛围 | 关闭战斗氛围条目，AI 不再用战斗语气 |
| 请求 AI 回复 | *（无需额外设置）* | 让 AI 描述逃跑的过程和之后的场景 |

> **逃跑不重置 HP。** 这是设计选择——如果玩家逃跑后再遇到战斗，HP 保持逃跑时的值。你也可以加一个「修改变量」动作把 HP 回满，取决于你想要的游戏体验。

#### 行为 4：敌人被击败

这个行为不是由按钮触发的，而是由**变量跨越阈值**自动触发。

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 名称 | 敌人被击败 | 给你自己看的 |
| 触发条件 | 选「变量跨越阈值」→ 变量 `enemy_hp`，方向「下降」，阈值 `0` | 当 enemy_hp 从正数降到 0 或以下时自动触发 |

执行动作：

| 动作类型 | 设置 | 作用 |
|---------|------|------|
| 显示通知 | 类型：成就，内容：`击败敌人！` | 在屏幕上弹出一个成就风格的通知 |
| 修改变量 | `in_combat` 设为 `false` | 退出战斗状态 |
| 禁用知识条目 | 战斗氛围 | 关闭战斗氛围条目 |

> **什么是「变量跨越阈值」？** 引擎会监控指定变量的值。当变量的值从阈值的一侧变到另一侧（这里是从 >0 降到 <=0），就会触发这个行为。注意：如果 enemy_hp 已经是 0 了，再减也不会重复触发——只有在**跨越**的那一刻才触发。
>
> **为什么没有「请求 AI 回复」？** 因为这个行为是被攻击行为间接触发的——攻击行为的最后已经有「请求 AI 回复」了。加上战斗氛围条目里有 HP 数据，AI 会自然地在回复中描述敌人倒下的场景。

#### 行为 5：玩家死亡

和行为 4 类似，也是变量跨越阈值触发。

| 字段 | 填什么 | 为什么这样填 |
|------|--------|-------------|
| 名称 | 玩家死亡 | 给你自己看的 |
| 触发条件 | 选「变量跨越阈值」→ 变量 `player_hp`，方向「下降」，阈值 `0` | 当 player_hp 降到 0 或以下时自动触发 |

执行动作：

| 动作类型 | 设置 | 作用 |
|---------|------|------|
| 显示通知 | 类型：危险，内容：`你死了` | 弹出一个红色危险风格的通知 |
| 修改变量 | `in_combat` 设为 `false` | 退出战斗状态，战斗 UI 消失 |
| 禁用知识条目 | 战斗氛围 | 关闭战斗氛围条目 |

> **通知类型的区别。** 「成就」通知是金色/绿色的，适合正面事件；「危险」通知是红色的，适合负面事件。选择合适的类型能让玩家的情绪体验更好。

::: tip 行为的执行顺序
当玩家点击「攻击」时，执行顺序是这样的：
1. 「攻击」行为执行 → enemy_hp 减 15，player_hp 减 10
2. 引擎检查所有变量跨越阈值的行为 → 如果 enemy_hp 跨过了 0，触发「敌人被击败」
3. 引擎检查所有变量跨越阈值的行为 → 如果 player_hp 跨过了 0，触发「玩家死亡」
4. 最后执行「请求 AI 回复」→ AI 根据最终状态写描述

所以不用担心执行顺序——引擎会先处理完所有变量变化和连锁行为，最后才让 AI 回复。
:::

### 第 4 步：（可选）创建进入战斗的行为

前面我们设置了 `in_combat` 默认为 `false`。那怎么进入战斗呢？有两种方式：

**方式 A：用按钮进入战斗**

如果你的世界里有探索阶段，可以在消息渲染器里加一个「战斗！」按钮，或者让某个剧情选择触发战斗。创建一个行为：

| 字段 | 填什么 |
|------|--------|
| 名称 | 进入战斗 |
| 触发条件 | 选「动作」→ 动作 ID 填 `start_combat` |

执行动作：

| 动作类型 | 设置 |
|---------|------|
| 修改变量 | `in_combat` 设为 `true` |
| 修改变量 | `enemy_hp` 设为 `80`（重置敌人血量） |
| 启用知识条目 | 战斗氛围 |
| 显示通知 | 类型：警告，内容：`遭遇敌人！战斗开始！` |
| 请求 AI 回复 | *（让 AI 描述敌人出现的场景）* |

**方式 B：默认就在战斗中**

如果你的世界本来就是一个纯战斗场景（比如竞技场），直接把 `in_combat` 的默认值改为 `true`，并在知识库里把「战斗氛围」条目默认启用。这样玩家一开始就在战斗中。

> 本配方后面的消息渲染器代码同时支持两种方式——它检查 `in_combat` 变量来决定是否显示战斗 UI。

### 第 5 步：做带血条和按钮的消息渲染器

这是让战斗界面出现在聊天中的关键步骤。

编辑器 → **消息渲染器** 标签页 → 选「自定义 TSX」→ 粘贴以下代码：

```tsx
export default function Renderer({ content, renderMarkdown, messageIndex }) {
  const api = useYumina();

  // ---- 读取变量 ----
  const playerHp = Number(api.variables.player_hp ?? 100);
  const playerMp = Number(api.variables.player_mp ?? 50);
  const enemyHp = Number(api.variables.enemy_hp ?? 80);
  const inCombat = api.variables.in_combat === true;

  // ---- 计算百分比（用于血条宽度）----
  const hpPercent = Math.max(0, Math.min(100, (playerHp / 100) * 100));
  const mpPercent = Math.max(0, Math.min(100, (playerMp / 50) * 100));
  const enemyHpPercent = Math.max(0, Math.min(100, (enemyHp / 80) * 100));

  // ---- 判断是否是最后一条消息 ----
  const msgs = api.messages || [];
  const isLastMsg = messageIndex === msgs.length - 1;

  // ---- 根据 HP 百分比选择颜色 ----
  const getHpColor = (percent) => {
    if (percent > 60) return "#22c55e";   // 绿色：血量充足
    if (percent > 30) return "#eab308";   // 黄色：血量不足
    return "#ef4444";                      // 红色：危险
  };

  return (
    <div>
      {/* ---- 正常渲染消息文字 ---- */}
      <div
        style={{ color: "#e2e8f0", lineHeight: 1.7 }}
        dangerouslySetInnerHTML={{ __html: renderMarkdown(content) }}
      />

      {/* ---- 战斗 UI（只在最后一条消息、且正在战斗时显示）---- */}
      {isLastMsg && inCombat && (
        <div style={{
          marginTop: "16px",
          padding: "16px",
          background: "rgba(15, 23, 42, 0.8)",
          borderRadius: "12px",
          border: "1px solid #334155",
        }}>

          {/* ---- 玩家状态栏 ---- */}
          <div style={{ marginBottom: "12px" }}>
            <div style={{
              fontSize: "13px", color: "#94a3b8", marginBottom: "4px",
              display: "flex", justifyContent: "space-between",
            }}>
              <span>玩家 HP</span>
              <span>{playerHp} / 100</span>
            </div>
            <div style={{
              height: "12px", background: "#1e293b", borderRadius: "6px",
              overflow: "hidden",
            }}>
              <div style={{
                height: "100%",
                width: `${hpPercent}%`,
                background: getHpColor(hpPercent),
                borderRadius: "6px",
                transition: "width 0.5s ease, background 0.5s ease",
              }} />
            </div>
          </div>

          {/* ---- 玩家 MP 栏 ---- */}
          <div style={{ marginBottom: "12px" }}>
            <div style={{
              fontSize: "13px", color: "#94a3b8", marginBottom: "4px",
              display: "flex", justifyContent: "space-between",
            }}>
              <span>玩家 MP</span>
              <span>{playerMp} / 50</span>
            </div>
            <div style={{
              height: "12px", background: "#1e293b", borderRadius: "6px",
              overflow: "hidden",
            }}>
              <div style={{
                height: "100%",
                width: `${mpPercent}%`,
                background: "#3b82f6",
                borderRadius: "6px",
                transition: "width 0.5s ease",
              }} />
            </div>
          </div>

          {/* ---- 敌人 HP 栏 ---- */}
          <div style={{ marginBottom: "16px" }}>
            <div style={{
              fontSize: "13px", color: "#94a3b8", marginBottom: "4px",
              display: "flex", justifyContent: "space-between",
            }}>
              <span>敌人 HP</span>
              <span>{enemyHp} / 80</span>
            </div>
            <div style={{
              height: "12px", background: "#1e293b", borderRadius: "6px",
              overflow: "hidden",
            }}>
              <div style={{
                height: "100%",
                width: `${enemyHpPercent}%`,
                background: getHpColor(enemyHpPercent),
                borderRadius: "6px",
                transition: "width 0.5s ease, background 0.5s ease",
              }} />
            </div>
          </div>

          {/* ---- 动作按钮 ---- */}
          <div style={{ display: "flex", gap: "8px" }}>
            {/* 攻击按钮 */}
            <button
              onClick={() => api.executeAction("attack")}
              style={{
                flex: 1,
                padding: "12px",
                background: "linear-gradient(135deg, #7f1d1d, #991b1b)",
                border: "1px solid #dc2626",
                borderRadius: "8px",
                color: "#fecaca",
                fontSize: "14px",
                fontWeight: "bold",
                cursor: "pointer",
              }}
            >
              攻击
            </button>

            {/* 防御按钮 */}
            <button
              onClick={() => api.executeAction("defend")}
              style={{
                flex: 1,
                padding: "12px",
                background: "linear-gradient(135deg, #1e3a5f, #1e40af)",
                border: "1px solid #3b82f6",
                borderRadius: "8px",
                color: "#bfdbfe",
                fontSize: "14px",
                fontWeight: "bold",
                cursor: "pointer",
              }}
            >
              防御
            </button>

            {/* 逃跑按钮 */}
            <button
              onClick={() => api.executeAction("flee")}
              style={{
                flex: 1,
                padding: "12px",
                background: "linear-gradient(135deg, #374151, #4b5563)",
                border: "1px solid #6b7280",
                borderRadius: "8px",
                color: "#d1d5db",
                fontSize: "14px",
                fontWeight: "bold",
                cursor: "pointer",
              }}
            >
              逃跑
            </button>
          </div>
        </div>
      )}
    </div>
  );
}
```

**代码逐行解释：**

- `const api = useYumina()` — 获取 Yumina 的 API，读变量、触发行为
- `Number(api.variables.player_hp ?? 100)` — 读取变量值并转为数字。`?? 100` 表示如果变量不存在就用默认值 100
- `inCombat = api.variables.in_combat === true` — 判断是否在战斗中。只有 `true` 时才显示战斗 UI
- `hpPercent = Math.max(0, Math.min(100, ...))` — 把百分比限制在 0~100 之间，防止血条溢出
- `getHpColor(percent)` — 根据 HP 百分比返回不同颜色：绿色（充足）→ 黄色（不足）→ 红色（危险）
- `isLastMsg` — 只在最后一条消息显示战斗 UI，避免每条消息都出现重复的血条和按钮
- `isLastMsg && inCombat` — 两个条件都满足才显示：是最后一条消息 **且** 正在战斗中
- `width: \`${hpPercent}%\`` — 血条的宽度用百分比控制，HP 越低血条越短
- `transition: "width 0.5s ease"` — 血条变化时有平滑的过渡动画，视觉效果更好
- `api.executeAction("attack")` — 触发我们第 3 步创建的「攻击」行为。按钮本身不做任何计算——所有逻辑都在行为规则里

### 第 6 步：保存并测试

1. 点击编辑器顶部的「保存」
2. 如果你用的是**方式 B**（默认在战斗中），直接开一个新会话就能看到战斗 UI
3. 如果你用的是**方式 A**（按钮触发战斗），先在消息渲染器里加一个「开始战斗」按钮（调用 `api.executeAction("start_combat")`），然后开新会话点击它
4. 你应该看到最后一条消息下方出现了 HP/MP 血条和三个按钮
5. 点击「攻击」——敌人 HP 减 15，你的 HP 减 10，AI 描述战斗
6. 点击「防御」——你的 HP 只减 5，MP 恢复 10
7. 多点几次攻击，把敌人 HP 打到 0 以下——弹出「击败敌人！」成就通知，战斗 UI 消失
8. 开新会话再测试一次，这次故意让自己的 HP 归零——弹出「你死了」危险通知

**如果遇到问题：**

| 现象 | 可能的原因 | 解决方法 |
|------|-----------|---------|
| 看不到血条和按钮 | `in_combat` 变量是 `false` | 如果用方式 B，确认默认值改成了 `true`；如果用方式 A，确认触发了 `start_combat` 行为 |
| 按钮点了没反应 | 行为的动作 ID 和代码不一致 | 检查行为的动作 ID 是否完全匹配（`attack`、`defend`、`flee`），注意大小写 |
| HP 没变化 | 「修改变量」动作没有正确配置 | 回到行为编辑器，检查每个「修改变量」动作的变量 ID 和操作（加/减/设为）是否正确 |
| 血条不动 | 消息渲染器代码有语法错误 | 检查消息渲染器底部的编译状态，应该显示绿色「OK」 |
| AI 没有描述战斗 | 缺少「请求 AI 回复」动作 | 确认攻击/防御/逃跑行为的最后一个动作是「请求 AI 回复」 |
| 通知没弹出 | 变量跨越阈值的行为配置有误 | 确认「敌人被击败」行为的触发条件是 `enemy_hp` 下降跨过 `0`，不是等于 0 |
| 敌人 HP 变成负数但没触发 | 阈值方向设错了 | 确认方向是「下降」，不是「上升」 |
| 战斗结束后 UI 没消失 | `in_combat` 没被设为 `false` | 检查「敌人被击败」和「玩家死亡」行为里是否有「修改变量 → in_combat 设为 false」 |
| 战斗氛围没切换 | 知识条目名称不匹配 | 检查行为里「启用/禁用知识条目」选择的是否就是你创建的那个条目 |

## 进阶扩展思路

掌握了基础战斗系统后，你可以在此基础上扩展更多功能：

- **多种敌人** — 加更多变量（`enemy_name`、`enemy_attack`），用宏 `{{enemy_name}}` 在条目里动态显示敌人名称。进入战斗时的行为设置不同的敌人属性。
- **技能系统** — 加一个「魔法攻击」按钮，消耗 MP 但造成更高伤害。在行为里先检查 MP 是否足够——如果不够，显示「MP 不足！」通知。
- **物品栏** — 加一个「使用药水」按钮，恢复 HP 但消耗物品数量变量。
- **连续战斗** — 在「敌人被击败」行为里不立刻退出战斗，而是重置 enemy_hp 并用「告诉 AI」动作告诉 AI「新的敌人出现了」。
- **Boss 战** — 设置更高的敌人 HP，加一个「Boss 怒气值」变量，敌人 HP 低时进入狂暴状态（切换另一个知识条目）。

## 速查表

| 你想做的事 | 怎么做 |
|-----------|--------|
| 显示 HP/MP 血条 | 消息渲染器里读变量、计算百分比、渲染 `<div>` 宽度 |
| 攻击/防御/逃跑按钮 | 消息渲染器按钮调 `api.executeAction("动作ID")`，行为里配置对应动作 |
| 扣血/回蓝 | 行为 → 执行动作 → 修改变量 → 减去/加上指定数值 |
| 敌人被击败时弹通知 | 行为触发条件选「变量跨越阈值」→ enemy_hp 下降过 0 → 显示通知（成就） |
| 玩家死亡时弹通知 | 行为触发条件选「变量跨越阈值」→ player_hp 下降过 0 → 显示通知（危险） |
| 战斗中切换 AI 风格 | 创建默认禁用的知识条目，进入战斗时「启用知识条目」，退出时「禁用知识条目」 |
| 让 AI 知道当前血量 | 在知识条目内容里用宏 `{{player_hp}}`、`{{enemy_hp}}` |
| 只在战斗中显示 UI | 消息渲染器里判断 `in_combat === true` |
| 只在最后一条消息显示 | 消息渲染器里判断 `isLastMsg`（`messageIndex === msgs.length - 1`） |
| 血条颜色随 HP 变化 | 写一个根据百分比返回颜色的函数（绿 → 黄 → 红） |
| 战斗结束后隐藏 UI | 在胜利/死亡行为里把 `in_combat` 设为 `false` |
